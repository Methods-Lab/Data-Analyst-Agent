Project Overview & Vision

Data Analyst Agent is a conversational machine-learning application that lets any user — technical or non-technical — upload a CSV or Excel dataset and immediately receive predictions, insights, and answers from a trained offline ML model. The user simply uploads a file, clicks Train Agent, and can then chat with the system in plain English to explore the data.

The Problem It Solves

Most analytics tools require users to know SQL, Python, or BI software. Small teams, students, and managers often have valuable spreadsheets but no way to ask questions of them without a data scientist. This project removes that barrier — it analyses any structured dataset and answers questions about it through a chat interface, while a real machine-learning model runs every prediction in the background.

Core Principles

Domain-agnostic
Works on any structured tabular dataset — sales, surveys, sensor data, customer records — without prior assumptions about the columns.
Scale-independent
Handles a few hundred rows or several hundred thousand rows with constant performance through chunked streaming.
Prediction-first
Real machine-learning predictions are the primary deliverable — every answer is backed by trained models.
Explainable
Every result is paired with SHAP-based feature importance and a plain-English explanation.
Project Outcome
A fully deployed web application where a user can upload any dataset, train a custom machine-learning model on it, ask questions in natural language, generate three types of professional visualizations, and produce time-series forecasts, regression predictions, and classification analyses — all in seconds.

System Architecture

The system is built as a three-layer pipeline. Data flows from the user interface down through the offline ML engine and back up through the response layer.

Layer 1 — User Interface
Streamlit Web Application
Three-column layout: file upload sidebar, central workspace with metrics & charts, right-hand chat panel.
↓
Layer 2 — Machine-Learning Engine · THE BRAIN
Scikit-learn AutoML Pipeline
Reads & structures the dataset, detects the target column, trains DecisionTree + RandomForest classifiers, computes SHAP feature importance, runs 5-fold cross-validation, exposes deductive & abductive reasoning, builds regression and time-series forecasting models on demand.
↓
Layer 3 — Conversational Response Layer
Natural-Language Explanation
Takes the ML model's structured numeric output and converts it into bullet-pointed plain-English answers for non-technical users.

Design Principle

Every prediction, every metric, every chart on screen is computed by the offline ML engine. The system is fully reproducible — given the same inputs, the same model produces the same numbers every time. The conversational layer adds polish to the delivery but never changes the underlying analysis.

Reproducibility
Models are persisted to disk after training (models.pkl, rules.json, feature_importances.pkl) so predictions can be re-run, audited, or exported at any time.

Section 05

Machine-Learning Engine

The offline ML engine is the analytical core of the application. Every prediction, every chart, every metric the user sees comes from these scikit-learn-based components. No external service is involved in any analytical computation.

01

Dataset Ingestion & Structuring

Reads CSV / Excel files in memory-safe chunks. Detects column types (numeric, categorical, datetime, ID), parses dates, handles encoding issues (UTF-8, Latin-1), flattens multi-sheet workbooks, and infers the most likely prediction target.

02

Data Cleaning & Preprocessing

Drops empty & ID columns, imputes missing numeric values with the median and categorical values with the mode, encodes string categories as integer codes via LabelEncoder, and produces a fully model-ready DataFrame.

03

Model Training — AutoML Pipeline

Trains a DecisionTreeClassifier (extracts human-readable IF-THEN rules) and a RandomForestClassifier (200 trees, parallel). Uses stratified train/test split with safe fallback when classes are imbalanced.

04

Cross-Validation & Overfitting Guard

Runs 5-fold cross-validation on a representative sample to produce a statistically sound generalisation accuracy estimate (mean ± standard deviation), independent of any single train/test split.

05

Feature Importance — SHAP & Permutation

Computes SHAP values on the winning model using TreeExplainer for both global feature ranking and per-prediction attribution. Falls back to permutation importance when SHAP is unavailable.

06

Deductive Reasoning — Forward Chaining

Given user-supplied feature values, traverses the IF-THEN rule base extracted from the trained DecisionTree and returns the best-matching prediction with confidence and supporting rule conditions.

07

Abductive Reasoning — Best-Explanation Inference

Given an observed outcome class, ranks the features most likely responsible for it using permutation importance and a Naive-Bayes BayesianAbducer for posterior probability hints.

08

Regression — On-Demand Training

Trains a RandomForestRegressor inline when the user wants to predict a numeric column. Reports the prediction value, training R², and the top contributing features.

09

Time-Series Forecasting

Uses LinearRegression over monthly-resampled date columns to project numeric values forward by N periods. Reports trend direction, average, and projected percentage change.

10

Visualization Engine

Builds correlation heatmaps, scatter plots of the most strongly correlated pair, and distribution histograms — all using pandas statistics & plotly rendering with the application's brand palette.

11

Text Retrieval Engine

For text-heavy datasets, builds a TfidfVectorizer index over the rows and uses cosine similarity to retrieve the most relevant document evidence for any user query.

AI Layer — Conversational Response

The AI layer plays a single, well-defined role: turning the ML engine's structured numeric output into plain-English answers a non-technical user can read. It never makes a prediction, never computes a statistic, and never decides what to predict. It is the "voice" of the analysis.

Specific Responsibilities

A

Translating ML output into natural language

When the ML model returns a prediction with confidence and feature importances, the AI layer writes a short bullet-pointed explanation of what it means in business terms.

B

Answering conversational chat messages

Handles greetings, follow-up questions, and clarifications. Always grounded in the actual data context (column names, distributions, percentages) provided by the ML engine.

C

Refusing fabricated answers

Strict anti-hallucination guardrails: if the user references a column, name, or concept that is not present in the dataset, the AI layer is forced to refuse and list the actual columns instead of inventing statistics.

D

Enforcing consistent formatting

Every analytical answer is rendered in the same structure: a bold headline sentence followed by three to four bullet points, each containing a specific number, column name, or percentage from the data.

Strict Boundary

Important Distinction
The AI layer never decides which predictions are possible, never trains a model, and never computes a number. Every option shown in the Predict dialog, every metric on the dashboard, every chart, and every importance ranking comes directly from the trained scikit-learn models. The AI only writes the surrounding prose.

Side-by-Side Responsibility Split

Machine-Learning Engine

  • Trains all models (Decision Tree, Random Forest, Linear & RF Regression)
  • Computes feature importance via SHAP and permutation
  • Runs 5-fold cross-validation
  • Forecasts numeric columns over time
  • Calculates correlations & statistics
  • Generates all visualizations
  • Performs deductive & abductive reasoning
  • Detects which prediction types the dataset supports

AI Layer

  • Writes bullet-pointed explanations
  • Handles greetings and conversational turns
  • Refuses to answer about non-existent columns
  • Adds business context to numeric results
  • Maintains plain-English tone for non-technical users
  • Formats consistently (headline + bullets)

End-to-End Workflow

A complete pass from raw upload to answered question — the sequence the application follows every time a new dataset is loaded.

1

File Upload

User drags a CSV or Excel file into the sidebar. The system streams it in 200,000-row chunks to stay memory-flat on large files.

2

Schema Detection & Cleaning

The ML engine inspects column types, drops ID and empty columns, imputes missing values, encodes categoricals, and emits a fully structured DataFrame.

3

Model Training

DecisionTree + RandomForest are trained on the cleaned data. Rules are extracted to rules.json; the trained models are serialised to models.pkl.

4

Validation & Explainability

5-fold cross-validation produces the generalisation accuracy. SHAP TreeExplainer computes per-feature contributions saved to feature_importances.pkl.

5

User Interaction

The workspace populates with metrics, an impact chart, and the chat panel. The user can click a pinned action or type a question.

6

Answer Generation

The ML engine computes the requested analysis. The conversational layer formats the result as a bullet-pointed explanation. The response appears in the chat panel within seconds.

End-to-end latency
On a typical 10,000-row sales dataset: upload < 1s, cleaning ~ 1s, training ~ 3–5s, visualizations < 1s, chat replies 1–2s. The entire pipeline runs locally on the deployment server with no external dependencies for the analytical steps.

Prediction Capabilities

When the user clicks Predict, the system inspects the dataset's actual columns and dynamically lists every prediction type it can perform. Nothing is hardcoded — the options change with every dataset.

Time-Series Forecast
For any numeric column with an associated date column, the system uses scikit-learn's LinearRegression on monthly-resampled values to project the column forward by a user-chosen number of months. Reports trend direction, monthly average, and projected percentage change.
Classification
Uses the already-trained DecisionTree + RandomForest. The user picks a class to understand (abductive analysis) or supplies feature values for a custom prediction (deductive analysis). Returns prediction, confidence, and supporting rule conditions.
Regression
For any numeric column, trains a RandomForestRegressor inline using all other numeric features. The user supplies known values; blanks default to the column median. Returns the predicted value, training R², and top contributing features.
Custom Free-Text
User describes a prediction in plain English. The system validates every referenced column name against the actual dataset, auto-corrects close matches via fuzzy matching, and rejects clearly out-of-context references with the list of available columns.

Visualization Capabilities

Visualisation is treated as a first-class feature of the exploratory data analysis (EDA) phase. All charts are computed by pandas correlation math and rendered with plotly using the application's brand palette.

Correlation Heatmap
Pairwise correlation between every numeric column shown on a coral-cream-sage diverging color scale from −1 to +1. Cells annotated with the exact correlation value.
Scatter Plot
Automatically picks the two numeric columns with the strongest absolute correlation in the dataset and renders their relationship with the signed correlation in the title.
Distribution Histogram
User selects any numeric column; the system renders its frequency distribution with 30 bins. Below the chart it reports average, median, min, and max.

Most Impactful Columns Dashboard

In addition to the on-demand visualisations, the main workspace always displays a horizontal bar chart of the top eight columns ranked by SHAP importance. This gives the user an immediate visual sense of what drives the model's predictions.

What Was Delivered

A fully functional, publicly deployed web application that demonstrates the complete machine-learning lifecycle on arbitrary tabular data. The system was built end-to-end: file ingestion, automated cleaning, dual-model training (decision tree + random forest), SHAP explainability, cross-validation, time-series forecasting, regression, classification, correlation analysis, and a conversational chat interface — all wrapped in a polished custom-themed Streamlit UI and deployed with continuous integration on Railway.

Educational Value

  • Demonstrates the complete supervised-learning workflow from raw data to predictions
  • Exposes both tree-based interpretable models (DecisionTree) and ensemble accuracy (RandomForest)
  • Implements SHAP and permutation importance — current state of practice in ML explainability
  • Applies cross-validation to demonstrate honest generalisation estimates
  • Shows real-world software engineering: deployment, version control, error handling, UI design

Why It Matters

The project bridges the gap between machine-learning theory and practical analytics. A non-technical user with a spreadsheet can now access the same modelling techniques that previously required a dedicated data scientist. The architecture is deliberately domain-agnostic so the same system works for sales data, customer surveys, sensor readings, or academic datasets without modification.

Final Note
The offline ML engine is the foundation of every analytical claim the application makes. The conversational layer is the polished surface that makes those claims accessible to anyone. Together they form a complete, reproducible, and explainable data-analysis system suitable for both academic evaluation and real-world use.
